## 재사용성을 고려한 컴포넌트 구분법

- 관심사의 분리로 같은 기능을하는 코드들만 모아 컴포넌트 파일로 관리한다.



### 재사용성이 좋은 컴포넌트

- 비즈니스 로직이 없다.
- 샅태값이 없다. 단 마우스 오버(mouse over)와 같은 ui 효과를 위한 상태값은 제외

- container에서 비즈니스 로직도 있고 상태값도 있지만
- component에는 UI 요소만 담긴 파일로 나누어 관리해준다.
